BunLinks & KhaeZip

Each converted image can create a BunLink file:

  • Small .sbbun.json files with a short sixbit:// ID.
  • Include image name, mime type, and base64 payload.
  • Designed to bundle inside KhaeZip™ Sovereign Edition archives.

Later, your own tools can read those JSON files to hydrate images on demand, using the short ID instead of copying long base64 blobs everywhere.

Note: Strip prefix applies to the “Data URL” output and BunLink payloads. <img> and CSS snippets always include the full data: prefix.

Using SixBit Bun™

  • 1. Drag in or click to choose images (PNG, JPG, GIF, WebP, SVG).
  • 2. (Optional) Set max width or JPEG quality. SVG can be minified.
  • 3. Choose output: Data URL, <img> tag, or CSS background.
  • 4. Click Convert All. Preview and base64/code appear for each image.
  • 5. Use Copy, Download, or BunLink on any card.
  • 6. BunLink saves a small .sbbun.json mapping file with a short ID + base64.
  • 7. Drop those BunLink files into KhaeZip™ and archive them for your sovereign asset vault.
  • 8. All work is local. Nothing is uploaded or shared.
Tip: Use Strip prefix when you want only the raw base64 for direct HTML/CSS logic.
